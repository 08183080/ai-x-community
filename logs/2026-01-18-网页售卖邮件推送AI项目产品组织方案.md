# 2026-01-18-网页售卖邮件推送AI项目产品组织方案

## 目标
把“在阿里云服务器上跑的：填 `emails.txt` → 每天推送 AI 项目邮件”的程序，组织成一个可在线售卖、开通后自动生效、体验丝滑的产品。

## 最小可卖（1 天内能上线的版本）
- **落地页**：卖点 + 示例邮件截图/样例 + 价格 + 常见问题 + 购买按钮
- **支付**：先用最省事的“支付链接/收款码”（支付宝/微信/Stripe/PayMaster 等）
- **收集邮箱**：支付成功页跳转到一个表单（或页面）让用户提交邮箱
- **人工/半自动开通**：你每天/每小时把新增邮箱追加到 `emails.txt` 并生效（先卖起来）
- **交付体验**：提交邮箱后立刻发“欢迎邮件 + 明天开始推送 + 退订方式”

适用：验证需求、快速成交、先不重构你现有程序。

## 体验丝滑（推荐的“可规模化”版本）
把 `emails.txt` 升级为一个“订阅表”，并用接口把“支付 → 开通”打通。

### 核心流程
1. **用户购买** → 支付成功回调（webhook）
2. **自动创建订阅**：状态=active，绑定邮箱、套餐、到期时间
3. **每天定时任务**：拉取所有 active 订阅 → 发送邮件
4. **自助管理页**：改邮箱、暂停、退订（必须有退订）

### 数据结构（最小字段）
- email
- status（active/paused/cancelled）
- plan（free/monthly/yearly）
- createdAt / expiresAt
- lastSentAt

### 技术落点（不限定技术栈）
- **前端**：一个“订阅/购买”页面 + “管理订阅”页面
- **后端**：`/api/subscribe`、`/api/webhook/payment`、`/api/unsubscribe`、`/api/me/subscription`
- **定时**：cron（服务器 crontab / 云定时 / Vercel cron 等）

## 商业与合规（别踩坑，省你很多售后）
- **邮件合规**：必须明确用户同意（opt-in），每封邮件带**退订链接**，避免被当垃圾邮件
- **隐私与数据**：写清楚你收集什么（邮箱）、用来做什么（推送）、如何删除
- **备案/域名**：若面向国内用户，域名/站点合规与 ICP 相关事项要提前考虑
- **内容预期**：说明频率、内容来源、是否含广告/推广、是否可定制

## 定价与套餐建议（简单就好）
- **试用**：7 天/或 3 封（降低购买门槛）
- **标准**：按月
- **年付**：打折（提高现金流）
- **企业**：多邮箱/团队订阅（后续再做）

## 你现在立刻要决定的 4 个问题（决定实现路线）
1. **面向人群**：个人创作者/开发者/企业？（决定文案与定价）
2. **支付方式**：国内（支付宝/微信）还是海外（Stripe）为主？
3. **交付承诺**：每天固定时间？可选主题/行业？是否支持中英文？
4. **退订机制**：准备做自助退订（推荐）还是邮件回复退订（不推荐）？

## 下一步建议（按优先级）
1. 先做“最小可卖”：落地页 + 支付 + 表单收集邮箱 + 欢迎邮件
2. 有 10–20 个付费用户后，再做自动化：支付回调 → 自动开通 → 自助管理/退订
