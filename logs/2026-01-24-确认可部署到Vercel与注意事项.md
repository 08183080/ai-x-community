## 结论

- 本项目 **可以部署到 Vercel**：根目录为静态 HTML（`index.html` 等），并包含 Vercel 支持的 `/api/*.js` Serverless Functions。

## 依据（快速核对点）

- `vercel.json`：`version: 2` + `cleanUrls: true`，适用于静态站点托管。
- `api/*.js`：采用 `module.exports = async (req, res) => { ... }` 的 Vercel Function 形式。
- `package.json`：包含 `@vercel/kv` / `bcryptjs` / `jsonwebtoken`，Vercel 会自动安装依赖。

## 部署必备环境变量（否则部分功能不可用/不安全）

- 论坛 KV（README 已写）：
  - `KV_REST_API_URL`
  - `KV_REST_API_TOKEN`
  - `FORUM_ADMIN_TOKEN`
- 登录鉴权（建议必配，避免使用默认弱密钥）：
  - `JWT_SECRET`

## 风险提醒（上线前建议处理）

- `index.html` 里存在前端直写的第三方 API Key（智谱 `open.bigmodel.cn`）。前端会被任何人看到，**不适合生产**。
  - 建议：把调用改为 `/api/xxx` 代理，并把 key 放到 Vercel 环境变量里（如 `ZHIPU_API_KEY`），前端只调用你自己的 `/api`。
