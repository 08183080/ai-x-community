## 新闻采集建议

### 每日检查清单

1. **早上**（9:00-10:00）
   - OpenAI、Google、Meta的官方博客
   - 主要科技媒体的AI板块

2. **中午**（12:00-13:00）
   - Hacker News热门AI话题
   - GitHub trending AI项目

3. **晚上**（18:00-19:00）
   - Reddit MachineLearning社区
   - arXiv新论文

### 新闻分类标准

**模型发布**
- 新的语言模型、图像模型发布
- 模型重大更新版本
- 开源模型发布

**科学研究**
- 学术突破
- 研究论文发布
- 实验室成果

**开源社区**
- 开源项目发布
- 开发工具更新
- 社区动态

**应用落地**
- AI产品发布
- 商业应用案例
- 行业解决方案

**AI安全**
- 安全研究
- 伦理讨论
- 政策法规

**开发工具**
- 开发框架更新
- API更新
- 编程工具

**行业动态**
- 投资融资
- 公司收购
- 市场分析

### 新闻标题模板

```
{公司/机构}发布{产品/模型名称}
{研究主题}取得新进展
{产品名称}全面升级
AI在{领域}取得突破性进展
{公司}开源{模型/工具}代码
{工具/服务}推出新功能
```

### 新闻摘要写作

好的摘要应该包含：
- **核心内容**（1-2句话）
- **影响或意义**（1句话）
- **关键数据或细节**（可选）

示例：
```
OpenAI今日正式发布GPT-5预览版，在推理能力和多模态理解方面取得重大突破。
新模型在多项基准测试中超越前代40%，并将向Plus用户逐步开放访问权限。
```

## 新闻更新流程

1. **收集阶段**
   - 浏览上述新闻源
   - 保存感兴趣的文章链接

2. **整理阶段**
   - 打开 news-editor.html
   - 填写新闻信息
   - 选择合适分类

3. **发布阶段**
   - 下载生成的 news.json
   - 替换原有文件
   - 刷新主页查看效果

## 保持新闻质量

### 选择标准

- **相关性**：与AI/ML/LLM直接相关
- **时效性**：优先发布3天内的新闻
- **权威性**：优先选择官方来源和知名媒体
- **价值性**：对社区有实际价值或启发性

### 避免内容

- 纯广告宣传
- 低质量营销内容
- 过时的新闻
- 未经证实的传闻
- 重复报道
